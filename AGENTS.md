# AGENTS.md

## Project Context

- 这是一个使用`hugo`配置的博客项目，你的任务是每次编写或修改一篇博客文章

## AI Agent 指令指南：工作流 SOP

Agent **必须** 遵循以下标准操作流程：

### Step 1: 确认目标

从给你的任务描述中提取出任务目标，比如是“撰写博客”还是“润色修改”等。

在这一步，你必须明确：

- 文章主题
- 文章分类（唯一，你可以使用命令`ls public/categories`查看已有的分类）
- 文章标签（一个或多个，你可以使用命令`ls public/tags`查看已有的标签）

### Step 2: 构建大纲

大纲不是非要遵循下面的规则，但是给你一些示例：

- 如果是教程类的文章
```
开篇：痛点场景引入
第一部分：为什么需要这个方法
第二部分：方法详解 - 分步
第三部分：实际案例演示（可选）
第四部分：常见问题/避坑指南（可选）
结尾
```
- 如果是全新的产品介绍类的文章
```
开篇：痛点场景引入
第一部分：为什么需要这个产品
第二部分：功能详解，解决什么问题 - 分步
第四部分：常见问题/避坑指南
结尾
```
- 如果是产品新功能介绍类的文章
```
开篇：产品介绍
第二部分：升级功能详解，解决什么问题 - 分步
第四部分：常见问题/避坑指南
结尾
```

### Step 3: 文章标题

你收到的任务一般会明确文章的标题，如果你觉得有必要，可以适当优化。如果未提供标题，根据文章主题和大纲，生成一个标题。标题原则：

- 尽量保持简洁但是要抓人眼球
- 绝对避免有“标题党”感觉的标题

标题生成后，创建一个`slug`: 由文章标题的简短英文翻译生成，全小写，单词之间用短横线相连。

### Step 4: 创建文件

你需要先获取这些变量：

- `year`: `YYYY`格式的当前年份
- `date`: `YYYY-MM-DD`格式的当前日期
- `slug`: 上一步生成的`slug`

调用命令`hugo new content/posts/{year}/{date}-{slug}.md`，文件创建完毕。修改`frontmatter`中的`title`属性为正确的标题，修改`tags`和`categories`属性为正确的分类和标签。

### Step 5: 填充正文

在写正文之前，先明确两件事：

- 文章的目标受众（一般是偏技术的用户和普通用户）
- 作者以往写作风格（你可以在`content/posts`目录下查看作者以往发布的文章了解其写作风格），作者不喜欢拐弯抹角，喜欢直达主题，喜欢用最平常的语气描述深奥的原理。

根据你列出的大纲，逐步填入正文。

### Step 6: 配图（可选）

如果需要配图，先生成`prompt`，然后调用工具：`node --env-file=.env scripts/genImage.js {prompt} -s /path/to/result.jpg`，这样就会生成图片到指定目录下。如果你想指定宽高比，添加`-r`参数，有如下比例可供选择：

- 1:1
- 16:9
- 9:16
- 2.35:1

假设文章路径为`content/posts/2026/2026-02-13-the-wenyan-family-ushers-in-a-major-update.md`，图片应该生成在`static/imgs/posts/2026-02-13-the-wenyan-family-ushers-in-a-major-update/4.jpg`，正文中应该插入`![](imgs/posts/2026-02-13-the-wenyan-family-ushers-in-a-major-update/4.jpg)`。注意相对路径中的`static`应该被清除。

### Step 7: 询问是否需要生成微信公众号封面

最后一步，询问作者是否需要生成微信公众号封面，如果选择“是”，则按照`Step 6`的流程，生成一张`2.35:1`的图片，并在`frontmatter`中添加一个属性：`cover: path/to/cover`
